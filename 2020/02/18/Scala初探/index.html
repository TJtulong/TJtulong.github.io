Scala概述

Scala是把面向对象与函数式编程结合在一起的高级语言。

Scala combines object-oriented and functional programming in one concise, high-level language. Scala’s static types help avoid bugs in complex applications, and its JVM and JavaScript runtimes let you build high-performance systems with easy access to huge ecosystems of libraries.

特点

  1. 面向对象编程

  2. 面向函数式编程

  3. 静态类型语言（变量类型编译阶段确定）
  4. 基于JVM （aa.scala -> aa.class -> JVM）

为什么使用Scala

  1. 速度快
  2. 优雅（代码量少）
  3. 融入大数据生态圈，Spark、Kafka是由Scala编写

安装

安装包下载：https://www.scala-lang.org/download/

直接解压安装包并配置环境变量：C:\Program Files\scala-2.13.1\bin

在cmd中：

scala> print("hello scala") hello scala

即为安装成功！

创建HelloWorld.scala代码：

scalac HelloWorld.scala

生成HelloWorld.Class文件，再去执行即可

scala HelloWorld.scala

构造函数

类自带有无参的构造函数，主构造函数会执行类中定义的所有语句。

分为主构造函数和辅助构造函数

单例对象与伴生对象

object，相当于Java中的工具类，可以定义工具函数和常量

和object类同名的class类称为伴生对象，可以互相访问彼此的私有属性和方法。

apply与unapply方法

apply方法又称注入器，用来构造对象

unapply方法又称提取器，用来提取属性

继承

Scala 使用 extends 关键字来继承一个类。实例中 Location 类继承了 Point 类。Point 称为父类(基类)，Location 称为子类。

override val xc 为重写了父类的字段。

继承会继承父类的所有属性和方法，Scala 只允许继承一个父类。

强制类型转换：

B.asInstanceOf[A]

抽象类

抽象类不能被实例化，可以有抽象方法和具体方法

特征

Scala Trait(特征) 相当于 Java 的接口，实际上它比接口还功能强大，与接口不同的是，它还可以定义属性和方法的实现。

一般情况下Scala的类只能够继承单一父类，但是如果是 Trait(特征) 的话就可以继承多个，从结果来看就是实现了多重继承。

样例类

使用了case关键字的类定义就是就是样例类(case classes)，样例类是种特殊的类，经过优化以用于模式匹配。

样例类的比较是基于值的比较而不是比较引用

模式匹配

一个模式匹配包含了一系列备选项，每个都开始于关键字 case。每个备选项都包含了一个模式及一到多个表达式。箭头符号 => 隔开了模式和表达式。

通配符模式

case List(_,_,3) //可以和List(1,2,3)匹配

Scala高级

高阶函数

参数或返回值为函数的函数称为高阶函数

闭包

闭包是一个函数，返回值依赖于声明在函数外部的一个或多个变量。

闭包通常来讲可以简单的认为是可以访问一个函数里面局部变量的另外一个函数。

柯里化

柯里化(Currying)指的是将原来接受两个参数的函数变成新的接受一个参数的函数的过程。新的函数返回一个以原有第二个参数为参数的函数。

方法的多态

Scala 中的方法可以按类型和值进行参数化。 语法和Java的泛型类类似。 类型参数括在方括号中，而值参数括在圆括号中。

Actor模型

Actor模型是一个并行计算模型，一切皆actor。

注：并发和并行

消息和邮箱（FIFO消息队列），接收消息放入Mailbox，并处理消息。

Actor的创建与使用

Scala提供了Actor trait来让我们更方便地进行actor多线程编程，就Actor trait就类似于Java中的Thread和Runnable一样，是基础的多线程基类和接口。

使用!符号，向actor发送异步消息，!?发送同步消息，!!发送带有返回值的消息
